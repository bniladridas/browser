# Codex Contributing Guide

## Scope

Use this file to capture how Codex should operate within this repository.

## Guidelines

- Keep changes small and directly tied to the request.
- Follow PR title format: `type[scope] :: description`.
- Follow commit format: `type(scope): description` or `type[scope]: description`.
- Do not bypass hooks or skip required checks.
- Avoid editing generated files unless explicitly requested.

## Where to Add Notes

- Conventions and decisions: `.codex/NOTES.md`.
- Repeatable tasks: `.codex/WORKFLOWS.md`.
- Templates and checklists: `.codex/TEMPLATES.md`, `.codex/CHECKLISTS.md`.
